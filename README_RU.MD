GLoader — компактный первичный установщик APK и помощник очистки для головного устройства Geely Citiray на Android 11 и новее. Приложение автоматически ищет `.apk` на подключённых съёмных USB-накопителях, показывает найденные файлы, позволяет выбрать APK вручную через системный файловый менеджер, показывает установленные приложения и умеет удалять приложения, ранее установленные через GLoader.

Интерфейс рассчитан как на экран телефона, включая Pixel 6 Pro, так и на большой вертикальный экран ГУ 13,2″. Проект использует только системные Android API, без тяжёлых библиотек, чтобы итоговый APK был небольшим и успевал установиться во время короткого окна доступности ADB.

## Первичная установка через ADB

Требуются `adb`, USB OTG и готовый APK из раздела [GitHub Releases](https://github.com/prodject/GLoader/releases).

```sh
chmod +x install.sh
./install.sh ./GLoader-1.X.apk
```

Скрипт ждёт появления устройства, быстро копирует APK в `/data/local/tmp`, выполняет `pm install`, пытается выдать доступ к внешнему хранилищу и разрешение установки APK через `appops`, удаляет временный файл и запускает GLoader. Результат применения `appops` зависит от прав ADB shell в конкретной прошивке ГУ.

## Использование

1. Подключите USB-накопитель с APK и запустите GLoader.
2. Откройте вкладку **Installer**. Приложение автоматически просканирует доступные съёмные тома.
3. Нажмите **Установить** у нужного APK либо **Выбрать APK** для ручного выбора.
4. При первом запуске Android может попросить разрешить доступ ко всем файлам и установку из этого источника.

Android не позволяет обычному пользовательскому приложению бесшумно устанавливать APK. Если GLoader не установлен как системное приложение или Device Owner, штатное подтверждение установки Android остаётся обязательным.

Вкладка **Apps** показывает установленные пакеты, видимые GLoader. Пользовательские приложения можно отправить в штатный сценарий удаления Android; защищённые системные пакеты отображаются как недоступные для удаления.

Вкладка **Cleanup** хранит список пакетов, успешно установленных через GLoader. Она может последовательно запросить удаление этих tracked-приложений, а затем при необходимости запросить удаление самого GLoader. Android запрашивает подтверждение для каждого удаления. При удалении GLoader Android также удаляет private app data самого GLoader.

Важное ограничение: обычное Android-приложение не может стереть системные записи Package Manager, историю установок или системные логи. Cleanup покрывает только tracked-приложения, запрос самоудаления GLoader и private app data GLoader, которые Android удаляет при uninstall.

## Сборка и релизы

GitHub Actions собирает минимизированный release APK при каждом запуске workflow на ветке `main` и публикует релиз `1.X`, где `X` — автоматически увеличиваемый номер запуска Actions. APK подписывается стандартным debug-ключом Android только для прямой установки; для публикации в магазине следует настроить собственную защищённую подпись.

Проект предназначен для сборки через включённый workflow GitHub Actions. Для локальной сборки, если она потребуется, необходимы JDK 17, Android SDK 35 и Gradle 8.9:

```sh
gradle assembleRelease
```
